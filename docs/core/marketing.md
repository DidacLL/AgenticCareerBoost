# Marketing strategy

## Audience

1. **Recruiters** — need fast proof of capability and role-fit
2. **Engineering peers** — value visible craft, systems thinking, tooling depth

## Channels (priority order)

1. **Repository** — canonical source; every artifact lives here first
2. **Website** — curated public mirror; recruiter landing page
3. **LinkedIn / social** — selective distribution of narrative artifacts

## Cadence rule

Every sprint closure should produce **at least one narrative candidate** suitable
for social distribution. Hotfixes are exempt.

## Artifact rules

- Public communication must support the technical identity
- Every visible output should strengthen role-fit and memorability
- Adapt per channel — do not clone identical content across platforms
- Evidence over adjectives: repos, commits, diagrams, case studies

## Anti-patterns

- Generic self-promotion
- AI-hype bandwagon posts
- Founder-LARP tone
- Content without linked evidence
- Duplicating the same text across every channel
