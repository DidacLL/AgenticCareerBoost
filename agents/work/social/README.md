# agents/work/social/

Platform-adapted public posts generated by the CommunityManager role and
social AutoAgents.

Intermediate planning cards, failed fragments, rejected drafts, stale files, and
poisoned repair artifacts are not durable authority merely because they exist in
this folder. Use them only when a current run contract explicitly promotes them.

When the user declares an artifact family stale, rejected, poisoned, or
superseded, agents may locate matching files by filename/path for cleanup or
exclusion, but must not read those files for requirements, examples, voice, or
acceptance criteria.

## Canonical plan

- [`plan.md`](plan.md) — maintained by `SocialMediaPlanner`, reviewed by
  `CommunityManager`

## Naming convention

```text
YYYY-MM-DD-<channel>-<slug>.md
```

Examples: `2026-04-20-linkedin-bootstrap-launch.md`, `2026-04-25-github-sprint1-recap.md`

## Source template

[`agents/rules/templates/community-output.md`](../../rules/templates/community-output.md)

Full public drafts must use the template checks, including the pre-draft prose
gate when required and sealed-context review.
