# S005 Authoritative Findings

## Source Status

These findings are binding constraints for S005. They come from the accepted
topic-by-topic analysis of the prior conversation and from the private writing
sample only as abstract voice evidence. No private sample lines, political
content, or external-thread details are reused here.

## 1. Scope Control

S005 is campaign architecture. It does not activate sprint state, run validation
commands, publish artifacts, create generated data, or modify deployment-adjacent
files. The work lives in social-planning files until the user approves a later
step.

## 2. Discarded Material

The previous S005 story architecture and post-blueprint approach is unusable.
It is not a draft to improve. It trained the wrong frame, so it must not be
treated as evidence, backlog, or a foundation.

Rejected material includes:

- the neat five-part campaign arc;
- old post names and reader-effect labels;
- final-post or draft-first planning;
- social copy that sounds like industry reassurance.

## 3. Rejected Comfort Language

The campaign must not lean on familiar AI-industry phrases about collaboration,
judgment, taste, system sharpening, or model-wrapper craft. Those phrases make
the project sound safer and cleaner than the lived evidence supports.

The specific danger is conceptual: soft language hides the destructive cost of
probabilistic systems when they produce plausible structure faster than humans
can audit it.

## 4. Core Thesis

The campaign thesis is sharper:

AI can be useful while being structurally dangerous. It is nondeterministic
machinery that predicts plausible continuations, silently drifts, floods the
workspace, and can turn logs, roles, tests, state, and instructions into failure
surfaces.

The engineering problem is designing workflows that survive that machinery.

## 5. Human Recovery

The human in this project was not a decorative approval gate. The human detected
silent damage, rejected poisoned artifacts, corrected operational drift, paid
the cost of excess generated material, and restored the intended direction.

That recovery burden is part of the story.

## 6. Agent Definition

An agent is broader than an LLM call. A deterministic script, scheduled cleanup,
regex pass, workflow, model loop, or coordinated system can be an agent.
Inference is one possible function inside an agent, not the essence of agency.

S005 should use that wider definition when discussing agentic systems.

## 7. Provider Harnesses

Closed-provider harnesses are part of the failure surface. Hidden prompts,
wrapper instructions, safety layers, tool policies, and product-level
orchestration can steer behavior beyond the visible user request.

The campaign should make that issue concrete through project experience rather
than abstract anti-vendor performance.

## 8. Failure Taxonomy

The useful failures include:

- drift from the initial intention;
- tests and validators becoming accidental architecture;
- silent blockers and overengineering loops;
- temporary patches becoming durable assumptions;
- logs and state being treated as authority;
- provider-added wrapper text widening scope;
- role files becoming writing styles;
- generated volume burying prior decisions;
- the user losing orientation inside too much generated material.

These are distinct failure classes. They should not be collapsed into a single
generic "failure post" idea.

## 9. Campaign Method

S005 must start from a concept room: what the public discussion needs, what the
project adds, what readers should infer, which tensions are worth dramatizing,
and which evidence makes each point inspectable.

Final text is disposable. The reasoning structure is the durable work.

## 10. Style Constraints

The target voice is direct, argumentative, technically literate, and willing to
name weak abstractions. It should avoid:

- symmetrical generated paragraphs;
- repeated contrast formulas;
- influencer cadence;
- fake vulnerability;
- taxonomy dumps;
- childish simplification;
- technical boredom;
- polished phrases that conceal the actual cost of the work.

## 11. Evidence Discipline

Repo evidence matters, but not every repo file is automatically authoritative.
State, logs, plans, and generated artifacts can be poisoned by failed runs.
Discarded work is discarded, not historical source material.

## 12. File Discipline

New campaign artifacts must follow the existing social naming convention and
avoid inventing convenient repository invariants. A file, directory, README, or
status marker should be required only when a real project contract depends on
it.

## 13. Validation Trap

Narrative planning must not trigger the validation cascade:

planning text -> status export -> static validation -> report build -> accidental
publication or deployment risk.

That sequence is itself a failure class for S005 to remember.

## 14. S005 Output Boundary

The next S005 output is a decision package with campaign-structure candidates.
It should not choose final post bodies, publishable wording, fixed cadence, or
external publication actions.
