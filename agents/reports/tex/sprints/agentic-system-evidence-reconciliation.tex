\chapter{Executive Summary}

\takeaway{This report records a bounded system-review/documentation run after a
large local refactor. The run fixed the live drift found in review, documented
intentional draft deletion, preserved formal evidence, and verified that project
literature and repository state describe the same agentic development trail.}

The trigger was a local-vs-remote review of a broad AgenticSystem refactor. The
review found that the main architecture direction was correct: internal agent
rules, state, tools, tests, and report sources now live under
\safePath{agents/}; public runtime and downloadable artifacts live under
\safePath{site/}; root is limited to platform and crawler entrypoints.

The review also found three small but real live inconsistencies and one evidence
decision that needed explicit documentation:

\begin{enumerate}
  \item \safePath{agents/rules/core/ci-rules.md} still described old docs-lint
    and Lychee config paths.
  \item \safePath{site/content/projects.json} still named
    \safePath{public-status.json}, although the generated public status file is
    now \safePath{site/data/status.json}.
  \item \safePath{agents/reports/tex/Makefile} still pointed humans to the old
    \safePath{content/reports/tex/} root.
  \item Discarded social/profile draft bodies were intentionally not restored;
    they are candidate outputs, not durable evidence or future steering rules.
  \item Browser validation exposed a direct clean-route render defect, which was
    fixed with root-absolute shell assets and pathname-aware pushState routing.
\end{enumerate}

This report exists because the project claim is stronger than "a pretty
prompt." AgenticCareerBoost is meant to be a repository-backed record of
agentic development: rules, failures, corrections, evidence, validation, and
public artifacts must line up.

\chapter{Authority Boundary}

\takeaway{Reports and state are evidence. They explain what happened. They do
not override current user scope or stable rules.}

The reconciliation uses the existing authority model:

\toprule
\textbf{Surface} & \textbf{Meaning} \\
\midrule
\endfirsthead
\toprule
\textbf{Surface} & \textbf{Meaning} \\
\midrule
\endhead
\safePath{agents/rules/**} & Authoritative behavior layer: core truth, workflows, roles, and templates. \\
\safePath{agents/state/**} & Evidence and status layer: logs, research, summaries, current state, and human queues. \\
\safePath{agents/reports/tex/**} & Formal source documents for human-facing proof artifacts. \\
\safePath{site/files/reports/**} & Published PDFs intended to remain visible in the repository and public site. \\
\safePath{agents/work/social/**} & Working social material; future drafts may live here, but discarded old draft bodies do not steer future copy. \\

\end{longtable}

The key correction is not deletion for its own sake. It is category discipline.
Historical logs, summaries, research, and LaTeX reports are retained as proof.
Old discarded profile and social drafts are not retained as reusable context
because they can poison future copy and were never approved publication facts.

\chapter{Public Literature Alignment}

\takeaway{The literature now states the same thing the repository does:
AgenticCareerBoost is a traceable system with evidence, not just polished
output.}

This reconciliation updates public and repository-facing indexes so the formal
report set includes this document alongside the guide, refactor retrospective,
and earlier sprint reports.

The aligned story is:

\begin{itemize}
  \item the refactor simplified active structure;
  \item the review found small live drift and one intentionally discarded draft
    class;
  \item the reconciliation fixed the live drift;
  \item the project preserved evidence while preventing stale drafts from
    steering future copy;
  \item tests and validation now guard the corrected shape.
\end{itemize}

That is the project lore in engineering form: an agentic run is not complete
when it produces an artifact. It is complete when the artifact, the trace, the
rules, the public surface, and the validation evidence all describe the same
thing.
