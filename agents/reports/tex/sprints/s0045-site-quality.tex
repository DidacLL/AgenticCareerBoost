\chapter{Corrected Defects}

\begin{itemize}
  \item Removed deployment-host literals from repository source and tightened
    validation so host values are derived by checks or runtime code.
  \item Corrected an overbroad placeholder remediation: non-site docs/state use
    real public URLs, while site metadata/navigation remains runtime-derived.
  \item Removed a committed static sitemap artifact that could not know the
    final deployment origin for both root and project-path Pages deployments.
  \item Converted canonical, Open Graph, Twitter, manifest start URL, and
    manifest scope behavior to runtime-derived or runtime-relative values.
  \item Replaced copied dashboard data with generated status export.
  \item Added dashboard runtime rendering from generated status JSON.
  \item Reworked the AgenticCareerBoost project page and dashboard route into a
    status dashboard with workflow, sprint, progress, closure, blocker, source,
    and evidence sections.
  \item Adjusted Contact, Projects, dashboard, and CV layouts for desktop and
    mobile readability.
  \item Added legal disclosure outside the OS panel/main document.
  \item Added explicit CV view state classes while preserving pushState
    filtering.
\end{itemize}

\chapter{Failure And Limit Ledger}

\begin{itemize}
  \item Deployment-host literals were too easy to leak into source, so runtime
    derivation and validation became necessary.
  \item Static source files cannot always know final deployment origin across
    root and project-path Pages deployments.
  \item Copying dashboard data was rejected in favor of generated status export.
  \item Render validation found layout defects after static checks had passed;
    browser checks were therefore essential, not decorative.
\end{itemize}
