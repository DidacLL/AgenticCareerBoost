\chapter{What Changed}

\section{Route Map}

The public route model was simplified to:

\begin{itemize}
  \item Home;
  \item Projects;
  \item Blog;
  \item CV;
  \item Contact;
  \item utility hire routes for ML, agentic systems, and backend contexts.
\end{itemize}

This reduced ambiguity. The old mix of Intro, Work, Notes, and Hire labels made
the site feel like an internal navigation experiment. The new route map gave
readers obvious public tasks: understand the person, inspect projects, read
notes, review a CV, and make contact.

\section{Project Detail Pages}

Project pages moved toward human explanations instead of internal template
proof language. The key copy pattern became:

\begin{enumerate}
  \item what the project is;
  \item why it was built;
  \item how it works;
  \item what was learned;
  \item where the source evidence lives.
\end{enumerate}

\section{Metadata And Data Surfaces}

S-003 added or revised public metadata and machine-readable surfaces:
\pathref{site/manifest.json}, \pathref{site/robots.txt},
\pathref{site/sitemap.xml}, blog index data, OS index data, and compatibility
metadata for notes.

\section{Interaction And Styling}

The monitor visual system was refined with subtler filters, scanlines, and dark
corner masking. JavaScript was updated for panel aliases, soft-navigation head
sync, and re-entrant CV initialization.
