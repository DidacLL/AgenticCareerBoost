## Code changes

- `.github/scripts/validate_static_site.py`
  - requires generated public status data;
  - verifies root and site status JSON match;
  - rejects deployment-bound host usage in site page/deploy source;
  - falls back to scanning `site/**` for artifact hygiene when `git ls-files` is unavailable;
  - rejects raw component `px` usage outside tokens and breakpoints;
  - validates runtime-relative manifest paths;
  - keeps deployed artifact hygiene checks for IDE metadata.
- `.github/scripts/export_status.py`
  - derives public dashboard data from `state/current.md`, `state/active-sprint.md`, and `state/roadmap.md`;
  - emits source pointers, sprint status, blockers, next sprint seed, and closure artifacts.
- `site/assets/css/site.css`
  - centralized semantic color/spacing/layout tokens;
  - converted component pixel sizing to tokens or relative units;
  - added responsive dashboard, contact-grid, compact preview-row, CV grid, and legal-disclosure styles.
- `site/assets/js/os.js`
  - derives site root from the loaded script URL;
  - fills runtime canonical/OG/Twitter metadata;
  - expands delegated soft navigation for internal HTML routes;
  - reloads dashboard/CV runtime scripts after soft navigation when needed.
- `site/assets/js/cv.js`
  - uses `history.pushState` and in-place filtering;
  - marks active controls and visible role sections for render/state gates.
- `site/assets/js/dashboard.js`
  - fetches generated status data from the current deployment root;
  - renders dashboard tiles, progress, artifact list, and blocker list.

## Generated-data flow

The committed source of truth stays in root state files. The deploy artifact `site/assets/data/public-status.json` is generated by CI/build steps and ignored in `site/.gitignore`, then validated against `data/public-status.json`.
