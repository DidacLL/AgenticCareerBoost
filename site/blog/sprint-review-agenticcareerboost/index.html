Sprint review

Cleaning a project is also project work.

Home / Blog / Sprint review

NOTE

The public surface was drifting

This sprint was not glamorous. Good. The useful work was mostly cleaning the places where the project had started to describe itself in different ways.

That kind of drift is easy to ignore because nothing explodes. The site still loads. The README still looks tidy. The state files still have dates and tables. But after a few rounds, each part starts telling a slightly different story. Then the next person, or the next agent, has to guess which version is real.

I do not want a public project that depends on guessing.

The root site moved to the clean GitHub Pages route. AgenticCareerBoost stayed as the source project behind it. That split matters: the public profile should be easy to open, while the workshop can keep the reports, state, workflow notes, and machinery.

The copy needed work too. A lot of it had the smell of generated positioning: symmetrical phrases, polite explanations, every paragraph carefully holding the reader's hand. It was not wrong in a factual sense. It was worse: it was soft.

So the cleanup became less about adding material and more about removing the voice that was not mine. The site should say what I work on, how I think about it, where the files are, and what kind of systems I trust. It does not need to bow after every sentence.

The remaining task is simple and annoying: keep the public surface and the repository state close enough that one does not become a costume for the other.
