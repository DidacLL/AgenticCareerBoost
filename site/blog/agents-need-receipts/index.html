Agents need receipts

Files remember more than a chat window.

Home / Blog / Agents need receipts

NOTE

Keep the work where it can be reopened

I like agentic work, but I do not trust the way people keep talking about it. Too much of the conversation sounds like a magic trick performed inside a private chat window. The result appears, everyone claps, and then next week nobody knows why the thing is shaped like that.

That is not a workflow. That is a séance with autocomplete.

If an agent changes a project, the project should show the change. Not as a screenshot, not as a beautiful explanation, not as a vague summary at the end. Files changed. Assumptions written down. Checks run. A human decision somewhere in the path. The boring trail is the part that lets the work survive the moment.

This is why I keep pushing work back into repositories. Source files, state files, reports, review notes, validation scripts, pull requests. It is not because I enjoy ceremony. I actually dislike ceremony. I just dislike unrecoverable work more.

The chat can help think. It can draft, compare, search, rewrite, inspect, and break a task into pieces. Fine. But the chat is not the project. The project is what remains when the session is gone and someone has to reopen the decision without the theatrical fog around it.

There is a strange arrogance in pretending that generated work needs less structure. It needs more. Not more meetings, not more corporate perfume, just stronger handles: names, paths, commits, tests, notes, boundaries.

When I say agents need receipts, I mean the work should leave enough marks to be questioned. A system that cannot be questioned is not advanced. It is just convenient in the most dangerous way.
