BLOG / TECHNICAL NOTE

Agents need receipts

Useful tools should leave evidence behind.

Home / Blog / Agents need receipts

NOTE

The short version

Assisted engineering work is getting easier to generate. That makes the boring parts more important.

Not the demos. The receipts.

If a tool changes a project, I want to know what changed, what was checked, what assumptions were made, and where the human decision happened.

My rule is simple: bounded changes, visible notes, tests before merge, and human approval before public output.

That does not make the tool less useful. It makes the work easier to trust.

CONTEXT

Why it belongs here

This note is the small version of a larger idea inside AgenticCareerBoost: assisted work should leave enough evidence to be checked later.

The longer project turns that idea into a repo structure: files, reports, state, checks, review gates, and a public site that can be inspected without asking a chat to remember its own history.
