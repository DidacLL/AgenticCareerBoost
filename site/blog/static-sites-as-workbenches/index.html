Home / Blog / Static sites as workbenches

NOTE

Why this site is intentionally simple

This site is static HTML, CSS, a small JavaScript layer, and a few JSON files.

That constraint is useful. A personal technical site should be cheap to host, easy to inspect, hard to break, and honest about what it is doing.

It also makes agent-assisted edits safer. The change is a diff. Routes can be validated. The browser can show the result. The repository keeps the memory.

TRADE-OFF

What the constraint buys

I lose a CMS, a component system, and some automatic publishing comfort. I gain a surface that can be read as files without a private tour through a build stack.

That makes the site a workbench: CV views, project notes, blog entries, JSON indexes, and manual corrections in one inspectable place.
