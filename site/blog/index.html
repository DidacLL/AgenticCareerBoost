Notes with a source behind them

This section is a shelf for writing connected to actual artifacts: projects, repo changes, technical decisions, and notes that are clearer here than buried inside a file tree.

I like short writing when it has a spine. A note should point somewhere: to a project, a source file, a sprint review, or a decision worth keeping visible.

RULE

How notes get here

A note needs an anchor: a project page, a source file, a sprint artifact, or an external publication. Otherwise it stays in the notebook until it has earned a real page.

The source list lives in blog-index.json. The boring file is part of the point: if the site says something exists, there should be a path to it.
