01B / PROJECT

What it is

P3CTeX is my LaTeX class and package work for formal reports, diagrams, listings, screenshots, and CV material.

The project keeps visual decisions attached to source commands, conventions, and repeatable builds. A document should not become an opaque PDF the moment it looks finished.

WHY

Why I built it

Academic reports, technical diagrams, code listings, and CV variants become fragile when every document is hand-polished as a one-off.

P3CTeX gives those documents shared structure: covers, localized labels, listings, figures, screenshots, page conventions, and defaults that can be reused without rebuilding the whole format by hand.

HOW

How it works

The repository separates class/package decisions from individual documents. It collects page structure, report conventions, diagram support, code listing styles, screenshots, and build habits.

The related p3cTeX-UMLST work extracts UML and listing behavior into smaller packages, which makes the system easier to reason about and less dependent on a single large file.

WHERE IT FITS

Why it is on this site

P3CTeX shows the tooling side of my profile: define conventions, automate repeatable parts, keep outputs inspectable, and make documentation a buildable artifact.

The CV, reports, diagrams, and sprint documents on this site use the same preference: source first, polish second, reproducibility always in view.

LEARNED

What it taught me

Document engineering is still engineering: naming, defaults, compatibility, boundaries, and maintenance pressure.

Polish without a build path is future debt with nicer typography.
