01B / PROJECT

Document machinery

P3CTeX is my LaTeX class and package work for formal reports, diagrams, listings, screenshots, and CV material.

I dislike documents that are only controlled from the outside. A PDF can be pretty and still be dead. I want the source, the commands, the defaults, and the build path close enough to fix the thing next month.

SCOPE

What it handles

Page structure, localized academic labels, code listings, screenshots, diagrams, covers, headers, report conventions, and repeatable CV/report builds.

The related p3cTeX-UMLST work splits UML and listing behavior out of the larger box. Smaller packages are less dramatic and much easier to distrust properly.

USE

Where it shows up

The CV, reports, diagrams, and sprint documents around this site come from the same instinct: do not let important documents become hand-polished artifacts with no memory.

Good typography is welcome. Rebuildability is not optional.
