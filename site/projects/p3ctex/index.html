01B / PROJECT

What it is

P3CTeX is my LaTeX class and package work. It started from the very practical need to make formal reports, diagrams, listings, and CV material repeatable instead of hand-polished every time.

WHY

Why I built it

I like visual work, but I dislike documents that only look controlled from the outside. P3CTeX lets the style live in source, so the same document can be rebuilt, checked, and adapted without losing its structure.

HOW

How it works

The project separates reusable class/package decisions from the actual documents. It covers page structure, diagrams, code listings, report conventions, and the boring defaults that make a document system usable twice.

LEARNED

What I learned

Document engineering is still engineering: naming, defaults, boundaries, compatibility, and a lot of tiny choices that only become visible when the next document needs to compile.
