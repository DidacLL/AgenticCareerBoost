<section>
  <h2>What It Proves</h2>
  <div class="grid">
    <article class="card"><h3>Workflow Architecture</h3><p>Role files, workflow contracts, state files, and review logs form a small operating model for bounded agent work.</p></article>
    <article class="card"><h3>Trace Discipline</h3><p>Planning, implementation, review, CI gates, and closure are written into repo-visible artifacts instead of disappearing into chat history.</p></article>
    <article class="card"><h3>Public Delivery</h3><p>The repository mirrors into this GitHub Pages site, PDF reports, status data, and recruiter-facing project pages.</p></article>

      <h2>How To Read It</h2>
      <p>Start with the route files and state files if you want the operating model. Start with the reports if you want the higher-level explanation. Start with this site if you are screening quickly for role fit.</p>
      <p>The important engineering claim is not that an agent wrote text. The claim is that the work is decomposed, scoped, reviewed, surfaced, and recoverable by path.</p>
